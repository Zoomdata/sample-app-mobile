Filter

Type:
  • object
Properties:
Name Type Description
path string

Field name being filtered. This is usually the original column name in the database and can be found on the Fields tab of the Zoomdata Web UI in the configuration section of your Data Source.
operation FilterOperation

Filter operation being performed. Refer to FilterOperation for possible values.
value string | number | Array.<string> | Array.<number>

Value to filter by. The operation used will determine the format the value should be in. For LT, GT, LE, GE value should be a number or a formatted date. For EQUALS or NOTEQUALS value should be a number, string or formatted date. For EQUALSI or TEXT_SEARCH value must be a string. For IN or NOTIN value must be an array. For BETWEEN value must be an array of 2 items. For AND and OR value must be an array of filters.
Example
// Numeric Filter
{
    path: 'usersentiment',
    operation: 'BETWEEN',
    value: [-0.5, 0.5]
}

// Attribute Filter
{
    path: 'userstate',
    operation: 'IN',
    value: ['California', 'New York', 'Virginia']
}
