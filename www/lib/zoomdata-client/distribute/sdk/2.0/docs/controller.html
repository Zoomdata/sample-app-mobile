Namespace: controller

controller

Used to control custom visualizations. The controller object is the principal object of the Visualization API. It is available in the Chart Studio environment. It provides properties and methods for receiving data and information from Zoomdata and for controlling a visualization.

Members

dataAccessors :object

The dataAccessors object is used to access any of the variables associated with the visualization. These variables are created in Chart Studio > Manage > Variables

Type:
  • object
Example
controller.dataAccessors["Group By"].setGroup(newGroup);
controller.dataAccessors["Y Metric"].setMetric(newMetric);

element :HTMLElement

This variable indicates the HTML element to which your visualization is attached on a webpage. Zoomdata provides a value for this variable when the visualization is embedded.

Type:
  • HTMLElement
Example
var svg = d3.select(controller.element).append("svg")
     .attr("width", "100%")
     .attr("height", "100%");

eventDispatcher :EventEmitter

The Dashboard Event Dispatcher. Used to provide communication between visualizations.

Type:
  • EventEmitter
Example
// 1st visualization
controller.eventDispatcher.emit('filters:add', filters); //Triggers the event

// 2nd visualization
controller.eventDispatcher.on('filters:add', addFilter); // Call addFilter function on event 'filters:add'

fields

Fields of the visualization's data source.

state

The State of your visualization's data stream from the Zoomdata server. You can use state.set() and state.get() to store and retrieve any key/value pairs related to the visualization that you want to preserve and make accessible to the visualization and its context. Values can be entire JSON objects. You do not need to stringify these objects.

Example
controller.state.set('font', 'Source Sans Pro'); //Set the value of a "font" key for the visualization.

registerInteractiveElements(input, options)

Works with HTML and SVG elements to make an element interactive. Used to create an interaction event when it is clicked or tapped. In Zoomdata environment, this event triggers the radial menu. In a third-party web app, this event can be captured and used to control this or other visualizations or elements on the page. Zoomdata provides methods for interacting with canvas-based charts. For more information about interacting with canvas-based charts, see controller#menu or controller#tooltip.

Parameters:
Name Type Description
input HTMLElement

HTML Element to make interactive. Can be an HTML or SVG element.
options object

Interactive element options.

Properties
Name Type Description
events array

Optional. Array of events which open Radial Menu. "Click" by default.
data function

Function returns the data object that this element represents.
menu object

Optional. Config of Radial Menu.

Properties
Name Type Description
items array

The list of items which will be shown in Radial Menu. Default Zoomdata items: ["Filter", "Filter All", "Unfilter All", "Info", "Zoom", "Trend", "Exclude"]. Custom items could be used.
droppable boolean

Optional. Turns on/off Droppable Acceptor. "False" by default.
tooltip object

Optional. Tooltip options for color adc content setting (see example below).
Example
node.enter().append("div")
     .each(function() {
         var el = this,
             item = d3.select(el);
         controller.registerInteractiveElements(el, {
             // Open Radial Menu on "mouseenter" event
             events: ['mouseenter'],
             data: function() {return item.data()[0]},
             menu: {
                 items: ["Info", "Zoom", "Trend", "Exclude", "Custom"]
                 // Set the action for custom item
                 "Custom" : function(){
                     console.log(this); // "this" keyword keeps the link to the registered element.
                 },
                 // Override "Zoom" button for custom functionality
                 "Zoom" : function(){
                     console.log("Zoom");
                 }
             },
             // Turn On Drop Acceptor.
             droppable: true,
             tooltip: {
                 color: function(){
                     return color;
                 },
                 data: function() {//If data field is defined tooltip will use this one for creating a tooltip
                     return el.__data__[0];
                 }
             }
         });
     });
