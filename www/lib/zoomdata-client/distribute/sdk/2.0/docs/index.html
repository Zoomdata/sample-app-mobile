Zoomdata Application Framework API Reference

The Zoomdata Application Framework allows a developer to create data driven web applications. The Application Framework consists of the Zoomdata client library together with documentation and samples to help you use it. The Zoomdata JavaScript client library contains an Embedding API and a Query API. The Embedding API is used to embed a visualization in a web application and to control some of the features of the visualization. The Query API is used to retrieve the data itself from a Zoomdata server. The main objects of the Framework are:

  • the client object, which you use for creating and managing other objects
  • the visualization object, which is created by the client.visualize and which manages a visualization
  • the query object, which is used to extract data from the Zoomdata server and which supplies data to visualizations

Embedding API

The Embedding API is designed to embed and control in a web application any of the visualizations that exist in a Zoomdata server. With the API, you can:

  • Embed a visualization in a web application
  • Resize the visualization
  • Make visualization elements interact with each other
  • Make visualizations interact with each other
  • Reconfigure a visualization's filters, groups, metrics, and timeframe
  • Control the colors, rulers, and grids applied to a visualization
  • Change the visualization from one chart template to another, such as from a bar graph to a pie chart

Using a visualization requires only a few simple steps:

  1. Create a security key
  2. Add relevant dependencies to the HTML code
  3. Create JSON objects to configure a Zoomdata client and the visualization itself
  4. Create the client
  5. Run the visualization

Query API

The Query API is designed to channel data from a Zoomdata server and return it in JSON objects that are easy to use. Common uses include:

  • Presenting data in tabular form
  • Presenting individual data points in text/numeric form
  • Performing further calculations on data before presenting
  • Creating application elements such as buttons or input fields based on data

Using a query object requires only a few simple steps:

  1. Create a security key
  2. Create JSON objects to configure a Zoomdata client and the query itself
  3. Create the client
  4. Run the query
  5. Harvest, process, and present the data
